Question 2.1
If we just use RUN pip install pandas without a version, Docker will grab whatever the newest version is at the exact moment the image is built. So, if a colleague tries to build my Dockerfile in a year, they might get a newer version of pandas where some functions have been changed or removed. This is a big problem for reproducibility because the code might suddenly crash or give totally different results for them, even though they used my exact same Dockerfile.

Question 2.2
It's better to share the Dockerfile (the recipe) because it lets other people see exactly how the environment was built step-by-step. If you just send the built image (the cake), it might run perfectly, but it's basically a black box—they won't know what specific commands or background libraries were used to make it. Sharing the Dockerfile makes the research transparent, letting others check the exact dependencies and easily modify the setup if they need to for their own work.
