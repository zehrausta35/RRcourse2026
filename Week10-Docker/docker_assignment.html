Docker Assignment

Based on: the in-class hello-docker exercise
Submit: push a single .md or .txt file to your forked course repository, in the same folder as the Docker materials

Part 1.Mechanics

Work in your hello-docker folder from class.

For each task, copy and paste from your terminal the commands you ran and their output. Wrap them in code blocks (triple backticks) so they stay readable.

💡 Reminder: every time you change the Dockerfile, you must rebuild the image with docker build -t hello-docker . before running it again. Editing the recipe does not change the cake until you bake again.

Task 1.1 — Change the Python version

Edit your Dockerfile so it uses Python 3.9 instead of 3.11. Rebuild and run.

Submit
  • The full terminal session (commands + output) showing docker build and docker run. The output should now say "Python 3.9 inside a container".

Task 1.2 — Break and fix the Dockerfile

Replace your hello.py with:

import sys, pandas
print(f"Python {sys.version_info.major}.{sys.version_info.minor}, pandas {pandas.__version__}")

Rebuild and run. It will fail. Read the error, then add one line to your Dockerfile to fix it. The fix must pin a specific version of pandas.

Submit
  • The terminal session showing the failed run and its error message.
  • The terminal session showing the successful run after your fix.
  • Your final Dockerfile pasted as text.

Part 2.Reproducibility judgment

Answer in 3–5 sentences each.

Question 2.1 — Why pin?

In Task 1.2 you pinned a specific version of pandas. Suppose you had instead written RUN pip install pandas (no version). Your Dockerfile would still build and run today. Why is this a reproducibility problem? What concretely could go wrong, and when?

Question 2.2 — Recipe or cake?

If you want a colleague to reproduce your work, you can either send them the built image (e.g. via Docker Hub) or send them only the Dockerfile and hello.py. Which is better for reproducible research, and why? Give one concrete reason beyond "easier" or "faster."

Submission

Push a single .md or .txt file to your forked course repository, in the same folder as the Docker materials.

The file should contain:

  • Your terminal sessions for Task 1.1 and Task 1.2 (commands + output, in code blocks).
  • Your final Dockerfile as text.
  • Your two written answers.

Filename: docker_assignment_<yourlastname>.md

A small meta-note: pasting plain text — not screenshots — is itself good practice for reproducible research. Text is searchable, diff-able, and copy-pasteable. Screenshots are not.
